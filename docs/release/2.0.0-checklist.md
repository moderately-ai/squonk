# Squonk 2.0.0 coordinated release checklist

This checklist coordinates the immutable `2.0.0` artifacts across crates.io, PyPI,
npm, and GitHub. Registry-specific details and rollback procedures remain in the
Rust, Python, and npm distribution runbooks beside this file.

## One-time registry setup

- [ ] The protected GitHub environments `crates-io`, `pypi`, and `npm` each have a
  required reviewer.
- [ ] The `crates-io` environment contains `CARGO_REGISTRY_TOKEN`, scoped to publish
  only `squonk` and `squonk-ast`.
- [ ] PyPI trusted publishing for project `squonk` names organization
  `moderately-ai`, repository `squonk`, workflow `release-python.yml`, and
  environment `pypi`.
- [ ] Existing npm packages trust organization `moderately-ai`, repository
  `squonk`, workflow `release-npm.yml`, environment `npm`, and allow
  `npm publish`.
- [ ] For the first publication of `@squonk-sql/quiltdb` and the eight native npm
  packages, the `npm` environment temporarily contains `NPM_BOOTSTRAP_TOKEN` as
  described in `npm-distribution.md`.

## Final release commit

- [ ] `Cargo.toml` and `crates/squonk-wasm/package.json` both resolve to `2.0.0`;
  Python derives the same version dynamically.
- [ ] `CHANGELOG.md` accurately describes the complete `v1.0.0..HEAD` user-facing
  diff and has the intended release date.
- [ ] `cargo xtask preflight` passes and both GitHub required checks are green on
  the exact commit.
- [ ] `release-crates.yml`, `release-python.yml`, and `release-npm.yml` each pass a
  manual `publish: false` rehearsal on that commit.
- [ ] crates.io, PyPI, and npm contain 1.0.0 but not the intended 2.0.0 updates.

## Freeze the source

Create one annotated tag; all registry publishers require this exact ref and reject
branches or surface-specific tags.

```sh
git tag -a v2.0.0 -m "Squonk 2.0.0"
git push origin v2.0.0
```

- [ ] The tag-triggered Rust workflow passes the `v2.0.0` semver baseline identity
  check and both crate dry-runs.
- [ ] The tag-triggered Python workflow builds and install-smokes all four wheels
  plus the sdist.
- [ ] The tag-triggered npm workflow builds all eight WASM facades and eight native
  packages, then passes every consumer, declaration, size, and manifest gate.

## Publish

1. [ ] Dispatch `release-crates.yml` from `v2.0.0` with `publish: true`; approve
   `crates-io`; confirm both sparse-index checksums.
2. [ ] Dispatch `release-python.yml` from `v2.0.0` with `publish: true`; approve
   `pypi`.
3. [ ] Dispatch `release-npm.yml` from `v2.0.0` with `bootstrap: true` and
   `publish: false`; approve `npm`. Configure trusted publishing on the nine new
   packages, then delete and revoke `NPM_BOOTSTRAP_TOKEN`.
4. [ ] Dispatch `release-npm.yml` again from `v2.0.0` with `publish: true` and
   `bootstrap: false`; approve `npm`.
5. [ ] Create the GitHub release from `v2.0.0` using the curated 2.0.0 changelog.

## Registry smoke tests

- [ ] Run `docs/release/smoke-test.sh 2.0.0` against crates.io.
- [ ] Install `squonk==2.0.0` in a clean Python 3.11 virtualenv and run
  `docs/release/smoke_python.py`.
- [ ] Install `squonk@2.0.0` and `@squonk-sql/quiltdb@2.0.0` in clean Node 22 and
  Node 24 projects; test native and `--no-addons` fallback paths.
- [ ] Confirm docs.rs builds both crates with all features and each registry page
  displays the correct README, license, repository, and provenance.
